# WezTerm 配置经验总结

## 关键知识点

### 1. 事件处理器中的 Pane 对象访问
**问题：** 在 `format-tab-title` 事件中，`tab.active_pane` 是 `PaneInformation` 对象，不是 `Pane` 对象。

**错误做法：**
```lua
-- 错误！PaneInformation 没有 get_domain_name 方法
local domain = tab.active_pane:get_domain_name()
```

**正确做法：**
```lua
-- 正确！需要通过 wezterm.mux.get_pane() 获取真正的 Pane 对象
local pane = wezterm.mux.get_pane(tab.active_pane.pane_id)
local domain = pane:get_domain_name()
local process = pane:get_foreground_process_name()
```

### 2. 标签标题截断问题
**问题：** 即使删除了 `tab_max_width` 配置，事件处理器中的 `max_width` 参数仍然存在，代码中手动截断会导致标题显示不全。

**解决方案：**
- 完全移除代码中的截断逻辑（`string.sub` 和 `…` 添加）
- 不要依赖 `max_width` 参数进行截断
- WezTerm 会自动处理标签宽度

### 3. 事件返回值格式
`format-tab-title` 可以返回：
- 字符串：纯文本标题
- FormatItem 数组：带颜色和样式的格式化标题

### 4. 模块导出格式
事件处理器必须包装在模块中并返回：
```lua
local M = {}
M.setup = function()
   wezterm.on('format-tab-title', function(...) ... end)
end
return M
```

不能直接在模块级别调用 `wezterm.on()`，否则 `require` 返回的是 `nil`。
